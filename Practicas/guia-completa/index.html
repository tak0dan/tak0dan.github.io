📚 1. Conceptos Fundamentales: HTML y CSS

¿Qué es HTML?

HTML (HyperText Markup Language) es el lenguaje de marcado que estructura el contenido de una página web. Define qué elementos hay en la página mediante etiquetas.

Estructura Básica de HTML

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8">
    <title>Mi Página</title>
  </head>
  <body>
    <h1>Contenido aquí</h1>
  </body>
</html>
Elemento: Es la unidad básica de HTML. Consiste en una etiqueta de apertura, contenido y etiqueta de cierre: <etiqueta>contenido</etiqueta>

Elementos Importantes

  • <div> - Contenedor genérico para agrupar elementos
  • <span> - Contenedor inline para texto
  • <header>, <main>, <footer> - Elementos semánticos
  • <section> - Define secciones temáticas
  • <article> - Contenido independiente
  • <nav> - Navegación

Listas HTML: ul, ol y li (profundidad de examen)

Las listas no son "solo bullets". Son estructura semantica y afectan accesibilidad, selectores y layout.

<ul>
    <li>Elemento sin orden</li>
    <li>Elemento sin orden</li>
</ul>

<ol>
    <li>Paso 1</li>
    <li>Paso 2</li>
</ol>
Matiz clave: li debe vivir dentro de ul o ol. Si lo sacas fuera, rompes semantica y algunos selectores como ul > li:last-child dejan de funcionar como esperas. 
/* Menu tipico con separadores */
.menu {
    display: flex;
    list-style: none;
    margin: 0;
    padding: 0;
}

.menu li:not(:last-child) {
    border-right: 2px solid red;
}

Tablas HTML: estructura correcta y errores sutiles

En tareas academicas, un table mal estructurado se detecta rapido aunque "se vea bien". Prioriza estructura correcta antes de estilo.

<table>
    <caption>Notas</caption>
    <thead>
        <tr>
            <th>Alumno</th>
            <th>Nota</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Nikita</td>
            <td>9</td>
        </tr>
    </tbody>
</table>
  • Usa th para cabeceras, no td.
  • Evita tablas para maquetar toda la pagina salvo que el ejercicio lo pida explicitamente.
  • Si el enunciado pide "exactamente como la muestra", respeta orden de filas/columnas y agrupaciones.

¿Qué es CSS?

CSS (Cascading Style Sheets) es el lenguaje que define la apariencia y disposición de los elementos HTML. Controla colores, tamaños, espacios, posiciones, etc.

Formas de Aplicar CSS

/* 1. CSS Inline (dentro del HTML) */
<h1 style="color: blue;">Título</h1>

/* 2. CSS Interno (en <style>) */
<style>
  h1 { color: blue; }
</style>

/* 3. CSS Externo (archivo .css) */
<link rel="stylesheet" href="estilos.css">
La Cascada de CSS: Los estilos se aplican en orden. El CSS externo es sobrescrito por el interno, que es sobrescrito por el inline. Los estilos más específicos sobrescriben los generales.

Selectores de CSS

/* Selectores básicos */
h1 { }              /* Elemento */
.clase { }          /* Clase */
#id { }             /* ID */
div.clase { }       /* Elemento con clase */
h1, h2, h3 { }      /* Múltiples selectores */

/* Selectores descendentes y hijos */
div p { }           /* Párrafos dentro de div */
div > p { }         /* Párrafos hijos directos de div */

/* Pseudoclases */
a:hover { }         /* Cuando el ratón está sobre el elemento */
button:active { }   /* Mientras se hace clic */
input:focus { }     /* Cuando tiene el foco */

Propiedades CSS Esenciales

/* Dimensiones */
width: 100px;
height: 200px;
max-width: 500px;
min-height: 100px;

/* Espacios */
margin: 20px;           /* Espacio FUERA del elemento */
padding: 15px;          /* Espacio DENTRO del elemento */
border: 2px solid #000; /* Borde */

/* Colores y Textos */
color: #ffffff;         /* Color del texto */
background-color: blue; /* Color de fondo */
font-size: 16px;
font-weight: 600;
text-align: center;

/* Posicionamiento (sin flexbox ni grid) */
position: relative;     /* Relativo a su posición normal */
position: absolute;     /* Relativo al padre posicionado */
position: fixed;        /* Relativo a la ventana */
top: 10px;
left: 20px;

El Modelo de Caja (Box Model)

Todo elemento en HTML es una caja rectangular. Entender el modelo de caja es fundamental:

content
padding
border
margin
/* De adentro hacia afuera:
   1. content (contenido real)
   2. padding (espacio interno)
   3. border (borde)
   4. margin (espacio externo)
*/

.box {
  width: 200px;           /* Ancho del contenido */
  padding: 15px;          /* 15px de espacio interno */
  border: 2px solid;      /* Borde de 2px */
  margin: 20px;           /* 20px de espacio externo */
  
  /* El box-sizing controla si el ancho incluye padding/border */
  box-sizing: border-box; /* Recomendado: ancho incluye padding y border */
}
⚠️ Importante: Por defecto, box-sizing: content-box hace que width sea solo el contenido. Es mejor usar box-sizing: border-box en todos los elementos (muchos developers lo hacen con * { box-sizing: border-box; }).

🎯 2. Flexbox: Layouts Modernos

Flexbox (Flexible Box Layout) es el sistema moderno para crear layouts. Es más flexible y potente que float, y es el método recomendado hoy en día.

Conceptos Clave de Flexbox

Contenedor Flex vs Items Flex

Contenedor Flex: El elemento padre que tiene display: flex. Define cómo se distribuyen los elementos hijos (items).
Items Flex: Los elementos hijos directos del contenedor. Se distribuyen según las propiedades del contenedor.

Propiedad: display: flex

/* Convertir un elemento en contenedor flex */
.container {
  display: flex;
  /* Ahora los hijos se distribuyen horizontalmente */
}
📌 En tus proyectos: Lo usaste en index.html (línea 43) para centrar contenido, y en Pr_3/5_paginas/css/estilos.css para crear layouts.

Propiedades del Contenedor Flex

flex-direction

Define la dirección en que se distribuyen los items (eje principal).

.container {
  display: flex;
  flex-direction: row;        /* Horizontal (por defecto) */
  /* flex-direction: column;   */ /* Vertical */
  /* flex-direction: row-reverse; */ /* Horizontal invertido */
  /* flex-direction: column-reverse; */ /* Vertical invertido */
}

Distribuye los items a lo largo del eje principal (horizontalmente si flex-direction: row).

  /* justify-content: center; */     /* Centrados */

  /* justify-content: space-around; */ /* Espacio uniforme alrededor */

gap

Define el espacio entre items (mucho más simple que usar margin).

.container {
  display: flex;
  gap: 16px;  /* Espacio de 16px entre todos los items */
  
  /* O controlar por separado */
  row-gap: 16px;    /* Espacio vertical */
  column-gap: 12px; /* Espacio horizontal */
}

Propiedades de los Items Flex

flex (propiedad abreviada)

Controla cómo se distribuye el espacio disponible entre items.

.item {
  flex: 1;  /* Crece para ocupar espacio disponible (proporción 1) */
}

/* Equivalente a: */
flex-grow: 1;      /* Cuánto crece si hay espacio extra */
flex-shrink: 1;    /* Cuánto se encoge si hay poco espacio */
flex-basis: auto;  /* Tamaño base antes de crecer/encogerse */
Ejemplo práctico: 
.container { display: flex; }
.item-1 { flex: 1; }   /* Ocupa 1/3 del espacio */
.item-2 { flex: 2; }   /* Ocupa 2/3 del espacio */
.item-3 { flex: 1; }   /* Ocupa 1/3 del espacio */
/* Total: 1+2+1 = 4 partes */

align-self

Alinea un item específico diferente a los demás.

.item-especial {
  align-self: center;   /* Este item se centra verticalmente */
  align-self: flex-end; /* Este item va al final */
}

order

Cambia el orden visual de los items sin cambiar el HTML.

.item-1 { order: 3; }  /* Aparece tercero */
.item-2 { order: 1; }  /* Aparece primero */
.item-3 { order: 2; }  /* Aparece segundo */

Nuances Profundas de Flexbox

1. El Eje Principal vs Eje Secundario

Flexbox opera sobre dos ejes:

  • Eje Principal: Definido por flex-direction. Si es row, es horizontal.
  • Eje Secundario (Cross Axis): Perpendicular al principal. Si el principal es horizontal, este es vertical.
/* Si flex-direction: row (horizontal) */
justify-content: ...;  /* Distribuye en EJE PRINCIPAL (horizontal) */
align-items: ...;      /* Distribuye en EJE SECUNDARIO (vertical) */

/* Si flex-direction: column (vertical) */
justify-content: ...;  /* Distribuye en EJE PRINCIPAL (vertical) */
align-items: ...;      /* Distribuye en EJE SECUNDARIO (horizontal) */

2. min-width y min-content

Los items flex no se comprimen por debajo de su min-width. Esto es importante para evitar que el contenido se rompa.

.container {
  display: flex;
  width: 300px;
  overflow: hidden; /* Evita que rebase */
}

.item {
  flex: 1;
  min-width: 100px;  /* No se comprimirá menos de 100px */
  /* Si el contenido es más ancho, rebasará */
}

3. Diferencia entre flex-basis y width

.item {
  width: 100px;        /* Tamaño fijo, pero flex lo puede cambiar */
  flex-basis: 100px;   /* Tamaño "base" antes de flexionar */
  flex: 1;             /* Crece si hay espacio */
}

/* Con flex, el width se ignora (flex-basis tiene precedencia) */

4. Collapsing Margins NO ocurre en Flexbox

En HTML normal, márgenes verticales se colapsan (toman el valor mayor). En flexbox, NO ocurre:

/* En display: block */
.item1 { margin-bottom: 20px; }
.item2 { margin-top: 30px; }
/* El espacio es 30px (solo el mayor) */

/* En display: flex */
.container { display: flex; flex-direction: column; }
.item1 { margin-bottom: 20px; }
.item2 { margin-top: 30px; }
/* El espacio es 50px (20px + 30px, no colapsa) */

5. Espacios Negativos (Negative Space)

.container {
  display: flex;
  gap: 16px;
}

.item {
  flex: 1;
  margin: -10px;  /* Márgenes negativos reducen el espacio */
}

🌊 3. Float: El Antiguo Sistema

Float es el sistema antiguo de layouts (pre-flexbox). Aunque obsoleto para layouts principales, es importante entenderlo porque:

  • Aún existe en código heredado
  • Es útil para envolver texto alrededor de imágenes
  • Puede ser más eficiente en algunos casos
📌 En tus proyectos: Usaste float en images/css/style.css (líneas 48-56): .content { float: left; width: 88%; } y .sidebar { float: right; width: 10%; }, y en practica5.1 con #imageLeft { float: left; } y #imageRight { float: right; }.

¿Cómo Funciona Float?

Float saca un elemento del flujo normal y lo coloca a un lado (izquierda o derecha), permitiendo que otros elementos fluyan alrededor de él.

/* Valores posibles */
.element {
  float: left;    /* Se desplaza a la izquierda */
  float: right;   /* Se desplaza a la derecha */
  float: none;    /* No flota (valor por defecto) */
}

Demo: Float Left

Contenido flotante a la izquierda
Este es el texto normal que fluye alrededor del elemento flotante. Cuando un elemento tiene float: left, se sale del flujo normal y los elementos siguientes lo rodean.

Demo: Float Right

Contenido flotante a la derecha
Similar al anterior, pero ahora el elemento está a la derecha. El texto fluye alrededor de él desde la izquierda.

Propiedad: clear

clear evita que un elemento tenga elementos flotantes a su lado.

.no-float {
  clear: left;    /* No permite flotantes a la izquierda */
  clear: right;   /* No permite flotantes a la derecha */
  clear: both;    /* No permite flotantes a ningún lado */
  clear: none;    /* Valor por defecto */
}
Caso de uso: Si tienes un <footer> después de elementos flotantes, necesitas clear: both para que no fluya alrededor del contenido flotante.

Problema: Collapse del Contenedor

Cuando todos los hijos de un contenedor están flotantes, el contenedor colapsa a altura 0. Soluciones:

/* Solución 1: overflow */
.container {
  overflow: hidden;  /* o auto */
  /* Esto "contiene" los elementos flotantes */
}

/* Solución 2: Clearfix (recomendado) */
.container::after {
  content: "";
  display: table;
  clear: both;
}

/* Solución 3: Hacer uso de display: flow-root */
.container {
  display: flow-root;  /* Solución moderna y limpia */
}

Nuances Profundas de Float

1. Elementos Flotantes No Respetan margin-collapse

Los márgenes verticales de elementos flotantes no colapsan (como en flexbox):

.float-left { 
  float: left; 
  margin-bottom: 20px;
}
.float-left { 
  margin-top: 30px; 
}
/* El espacio es 50px, no 30px */

2. Elementos Flotantes No Crean un Contexto de Apilamiento (Stacking Context)

Pero esto ha sido debilitado. Z-index puede funcionar de manera impredecible.

3. Float y Text-align

/* text-align NO afecta a elementos flotantes */
.container {
  text-align: center;
}

.container > img {
  float: left;  /* La imagen seguirá en la izquierda, no se alinea al centro */
}

4. Ancho 100% de Floats

/* Problema: ancho 100% con floats crea overflow */
.container {
  width: 400px;
}

.left {
  float: left;
  width: 50%;   /* 200px */
}

.right {
  float: left;
  width: 50%;   /* 200px */
  /* Total: 400px ✓ Funciona */
}

/* PERO si añades bordes o padding, rebasan */
.left {
  float: left;
  width: 50%;
  border: 1px solid;  /* +2px por el borde */
  /* Total: 402px ✗ Problema */
}

5. Float No Trabaja con Altura (Height)

/* height EN ELEMENTOS FLOTANTES es complicado */
.sidebar {
  float: right;
  width: 30%;
  height: 100%;  /* ✗ NO hace lo que esperas */
  /* Para altura 100%, necesitas height del padre */
}

.container {
  height: 500px;
}

.sidebar {
  float: right;
  width: 30%;
  height: 100%; /* ✓ Ahora y si, es 500px */
}

6. Diferencia entre width % y width px

/* Con floats, el % es relativo al padre */
.container { width: 600px; }
.sidebar { float: right; width: 50%; }  /* 300px */

/* Con width px, es directo */
.sidebar { float: right; width: 300px; }

📋 4. Formularios HTML

Los formularios son esenciales para recopilar datos del usuario. HTML proporciona muchos elementos de formulario.

📌 En tus proyectos: Tienes una práctica completa en Practica_formulario/formulario.html con inputs, textareas, radio buttons, checkboxes y selects.

Elemento: <form>

Contenedor que agrupa campos de formulario. Permite enviar datos a un servidor.

<form action="procesar.php" method="POST">
  <!-- Campos del formulario aquí -->
  <button type="submit">Enviar</button>
</form>

/* Atributos importantes */
action       /* URL a donde se envían los datos */
method       /* GET o POST (cómo se envían) */
name         /* Nombre del formulario */
target       /* Dónde abrir la respuesta (_blank, _self, etc.) */

Estructura de forma/formulario exacta (tipo examen)

Cuando piden "formulario de datos" como plantilla fija, el orden estructural suele evaluarse: primero fieldsets principales, luego agrupaciones secundarias, luego acciones.

<form>
    <fieldset>...datos personales...</fieldset>
    <fieldset>...idioma...</fieldset>
    <fieldset>...programas...</fieldset>
    <fieldset>...sistema operativo...</fieldset>
    <input type="reset">
</form>

Un cambio de orden o mover inputs fuera del fieldset puede invalidar la correccion aunque el estilo sea parecido.

GET vs POST

GET Datos en la URL. Visible. Limitación de tamaño. Inseguro para datos sensibles.
POST Datos en el cuerpo. Oculto. Sin límite de tamaño. Más seguro.

Elementos de Formulario

1. Input de Texto

<label for="nombre">Nombre:</label>
<input type="text" id="nombre" name="nombre" placeholder="Tu nombre" required>

2. Input de Email

<label for="email">Email:</label>
<input type="email" id="email" name="email" required>

3. Input de Contraseña

<label for="pass">Contraseña:</label>
<input type="password" id="pass" name="pass" required>

4. Textarea (Texto Multilinea)

<label for="comentarios">Comentarios:</label>
<textarea id="comentarios" name="comentarios" rows="4" placeholder="Escribe aquí..."></textarea>

Textfields con scroll (textarea scrollers)

Para textos largos, la tecnica clasica es fijar altura y activar scroll vertical. Esto aparece en practicas tipo tattoo/news cuando el texto debe quedarse dentro de una caja fija.

.text {
    height: 100%;
    overflow-y: auto;   /* Scroll vertical cuando excede */
    overflow-x: hidden; /* Evita scroll horizontal feo */
}

textarea {
    min-height: 100px;
    resize: vertical;   /* Permite agrandar solo en vertical */
    overflow-y: auto;
}
Error sutil comun: poner overflow-y: auto sin una altura concreta en el contenedor. Sin altura limite, no aparece scroll porque el bloque sigue creciendo.

5. Radio Buttons (Una opción)

<label>Género:</label>
<input type="radio" name="genero" value="masculino" id="m">
<label for="m">Masculino</label>

<input type="radio" name="genero" value="femenino" id="f">
<label for="f">Femenino</label>

<input type="radio" name="genero" value="otro" id="o">
<label for="o">Otro</label>

6. Checkboxes (Múltiples Opciones)

<label>Intereses:</label>
<input type="checkbox" name="deportes" value="futbol" id="futbol">
<label for="futbol">Fútbol</label>

<input type="checkbox" name="deportes" value="basquet" id="basquet">
<label for="basquet">Basquetbol</label>

7. Select (Lista Desplegable)

<label for="pais">País:</label>
<select id="pais" name="pais">
  <option value="">Selecciona un país</option>
  <option value="spain">España</option>
  <option value="mexico">México</option>
  <option value="argentina">Argentina</option>
</select>

8. Fieldset y Legend

Agrupan campos relacionados visualmente.

<fieldset>
  <legend>Datos Personales</legend>
  <input type="text" placeholder="Nombre">
  <input type="email" placeholder="Email">
</fieldset>

Atributos Importantes

required Campo obligatorio
placeholder Texto de ejemplo que desaparece al escribir
disabled Campo deshabilitado (no se puede usar)
readonly Campo de solo lectura (se ve pero no se puede editar)
maxlength Número máximo de caracteres

Validación HTML5

HTML5 proporciona validación integrada:

<input type="email" required>    /* Valida que sea email */
<input type="number" min="1" max="100">  /* Solo números entre 1-100 */
<input type="date">                     /* Selector de fecha */
<input type="time">                     /* Selector de hora */
<input type="url">                      /* Valida URL */
<input type="tel">                      /* Números de teléfono */

Nuances de Formularios

1. El Atributo "for" de Label

/* CORRECTO: El "for" coincide con el "id" del input */
<label for="nombre">Nombre:</label>
<input type="text" id="nombre">

/* INCORRECTO: No coinciden */
<label for="nombre">Nombre:</label>
<input type="text" id="nombre-input">

2. Radio Buttons: El Atributo "name"

Todos los radio buttons que deben ser exclusivos deben tener el MISMO name:

/* CORRECTO: Todos tienen name="color" */
<input type="radio" name="color" value="red">
<input type="radio" name="color" value="blue">
/* Solo se puede seleccionar uno */

/* INCORRECTO: Diferentes names */
<input type="radio" name="color1" value="red">
<input type="radio" name="color2" value="blue">
/* Se pueden seleccionar ambos (¡no debería pasar!) */

3. Checkboxes: Envío de Datos

/* Si un checkbox NO está marcado, NO se envía */
<form method="POST">
  <input type="checkbox" name="terminos">
  <button type="submit">Enviar</button>
</form>

/* Si se marca, se envía como: terminos=on
   Si NO se marca, NO se envía en el POST */

4. Input Type vs Value

/* type define QUÉ es el campo */
<input type="number">    /* Solo acepta números */
<input type="email">     /* Valida formato email */

/* value es el CONTENIDO inicial */
<input type="text" value="Hello">  /* Comienza con "Hello" */

📊 5. Gráficos y Visualización de Datos

Hay varias formas de crear gráficos en web. Las principales son:

1. Canvas (Dibujo Programático)

Canvas permite dibujar gráficos pixel por pixel usando JavaScript.

<canvas id="miCanvas" width="400" height="300"></canvas>

<script>
  const canvas = document.getElementById('miCanvas');
  const ctx = canvas.getContext('2d');
  
  /* Dibujar un rectángulo */
  ctx.fillStyle = '#7ad1a2';
  ctx.fillRect(50, 50, 200, 100);
  
  /* Dibujar un círculo */
  ctx.beginPath();
  ctx.arc(300, 150, 50, 0, Math.PI * 2);
  ctx.fillStyle = '#ff6a00';
  ctx.fill();
  
  /* Dibujar texto */
  ctx.fillStyle = '#ffffff';
  ctx.font = '16px Arial';
  ctx.fillText('Mi Gráfico', 10, 30);
</script>

Demo: Gráfico de Barras Simple

Demo: Gráfico Circular (Pie Chart)

2. SVG (Gráficos Vectoriales)

SVG es ideal para gráficos escalables que no pierden calidad.

<svg width="200" height="200" viewBox="0 0 200 200">
  <!-- Rectángulo -->
  <rect x="20" y="20" width="100" height="60" fill="#7ad1a2" />
  
  <!-- Círculo -->
  <circle cx="150" cy="100" r="40" fill="#ff6a00" />
  
  <!-- Línea -->
  <line x1="10" y1="10" x2="190" y2="190" stroke="#fff" stroke-width="2" />
  
  <!-- Texto -->
  <text x="50" y="150" font-size="16" fill="#fff">SVG Gráfico</text>
</svg>

Ventajas de SVG

  • Escalable sin pérdida de calidad
  • Se puede editar con CSS y JavaScript
  • Más accesible que Canvas (se puede usar con lectores de pantalla)

6.1. Layout desde cero: block, inline e inline-block

En Material/positions/positions se comparan visualmente los modos de caja base. Esta distincion es crucial porque muchos errores de layout vienen de no saber en que flujo vive cada elemento.

/* Block: ocupa toda la linea */
.box-block { display: block; width: 100%; }

/* Inline: fluye con texto, width/height no aplican como esperas */
.box-inline { display: inline; }

/* Inline-block: fluye en linea, pero acepta width/height */
.box-inline-block { display: inline-block; width: 220px; }

6.2. Position completo: static, relative, absolute, fixed, sticky

Los ejemplos absoluto.html, absolutoPadre.html, fijo.html y relativo.html muestran la regla clave:

Regla madre: un hijo position: absolute se posiciona respecto al ancestro mas cercano con position distinto de static. Si no existe, toma el viewport/documento como referencia. 
.padre {
    position: relative;

.hijo {
    position: absolute;
    top: 20px;
    left: 500px;

.flota {
    float: left;
    margin-right: 1em;
    margin-bottom: 1em;
}

.limpia {
    clear: both;
}

Tambien aparece la alternativa moderna para contener floats sin hacks:

.contenedor {
    display: flow-root; /* crea un nuevo bloque de formato */
}

6.4. Flexbox microdetallado (material flexbox.zip)

Ademas de lo anterior, el material extra incluye demos especificas de align-content, inline-flex, order, flex-basis, flex-shrink, flex-wrap y animaciones por transition.

.container {
    display: flex;
    flex-wrap: wrap;
    align-content: flex-start; /* controla lineas, no items individuales */
    height: 60vh;              /* necesario para notar align-content */
}

.item-a { flex-basis: 200px; }
.item-b { flex-shrink: 0; }
.item-c { order: 30; }

.animado {
    transition: all 1s;
}
.animado:hover {
    flex-basis: 400px;
}
Matiz critico: align-items alinea items dentro de cada linea; align-content alinea el conjunto de lineas. Si no hay multiples lineas (wrap) o no hay espacio libre en eje cruzado, align-content parece "no funcionar".

6.5. Selectores, especificidad y pseudoselectores (material examen)

En some_bullshit_pages/selectors.html e index.html del material de examen se documentan tipos de selectores y jerarquia de especificidad.

/* Escala tipica de especificidad */
/* inline (1000) > #id (100) > .class/[attr]/:pseudo-class (10) > element/::pseudo-element (1) */

#unique .spec-demo__text { color: #0b84ff; }
.spec-demo__text:hover { color: #ffb000; }
div .spec-demo__text { color: #f8f8f8; }

Tambien se refuerza el uso de:

  • [attr] y [attr="valor"]
  • Combinadores: div > p, div p, h2 + p, h2 ~ p

  • Pseudoelementos: ::before, ::after, ::first-letter

6.8. JavaScript aplicado a HTML/CSS: DOM, eventos y estado

El material tareas/js/tareas08.js, cesar/js/app.js y structure.js añade la capa de comportamiento:

// Esperar a que el DOM exista
document.addEventListener('DOMContentLoaded', () => {
    const form = document.querySelector('#formTarea');

    form.addEventListener('submit', (e) => {
        e.preventDefault();
        // leer datos, actualizar estado y repintar
    });
});

Patrones clave observados:

  • Estado en memoria: arrays de objetos (ej. tareas con id, prioridad, fecha, completada).
  • Renderizado: generar DOM/HTML desde estado y volver a pintar tras cada accion.
  • Delegacion de eventos: un solo listener en contenedor usando data-id y data-action.
  • Interaccion incremental: input event para sincronizar sliders en tiempo real.

🕵️ 7. Errores Sutiles y Estructura

Esta seccion esta pensada para ejercicios de examen donde la solucion parece minima, pero exige comprension profunda de estructura HTML, cascada CSS y flujo de layout. Muchos de estos ejercicios se resuelven cambiando una sola etiqueta o una sola regla.

7.1. Metodo forense para depurar rapido

  1. Verifica estructura base: doctype, html, head, body.
  2. Comprueba jerarquia de contenedores: wrappers, columnas, bloques principales.
  3. Inspecciona selectores que no aplican: clase mal escrita, id incorrecto, combinador equivocado.
  4. Revisa flujo de layout: block/inline/flex/grid/float y sus ejes.
  5. Analiza especificidad y orden en cascada antes de tocar !important.
  6. Solo despues corrige visualmente espaciados y colores.
Regla de oro: si algo "se ve mal", primero asume un error estructural, no decorativo.

7.2. Caso clasico: formulario que debe quedar EXACTO

En estos ejercicios, la estructura manda. Un formulario puede romperse por detalles aparentemente pequenos:

  • Un label sin for correcto.
  • Un input fuera de su fieldset.
  • Orden incorrecto de bloques (primero datos personales, luego opciones).
  • Falta de name en radios (rompe exclusividad).
<fieldset>
    <legend>Datos personales</legend>
    <label for="nombre">Nombre</label>
    <input id="nombre" name="nombre" type="text" required>
</fieldset>

<fieldset>
    <legend>Idioma</legend>
    <input type="radio" id="ingles1" name="ingles" value="1">
    <label for="ingles1">Ingles nivel 1</label>
</fieldset>

7.3. Caso clasico: HTML pequeno + CSS enorme e ilegible

Cuando hay miles de lineas de CSS, un solo div faltante rompe toda la maqueta. El patron tipico:

<div class="page">
    <div class="layout">
        <main class="content">...</main>
        <aside class="sidebar">...</aside>
    </div>
</div>

Si falta .layout, los selectores como .layout > .content o reglas flex/grid no aplican. Resultado: todo "parece" roto aunque el CSS este bien.

7.4. Caso clasico: lineas rojas entre items del menu

Objetivo: separadores rojos entre items sin linea al final.

.menu {
    display: flex;
    list-style: none;
    margin: 0;
    padding: 0;
}

.menu li {
    padding: 0 12px;
    border-right: 2px solid red;
}

.menu li:last-child {
    border-right: none;
}

Error sutil comun: aplicar el borde al a en vez del li y luego pelear con alturas/line-height.

7.5. Caso clasico: "el CSS rehace todo el HTML"

Un mismo HTML puede parecer totalmente distinto segun el modo de layout:

  • Normal flow: bloques uno debajo de otro.
  • Flex: distribucion en un eje, alineacion y reordenado visual.
  • Grid: malla 2D que puede cambiar orden visual y areas.
  • Position: elementos sacados de flujo para overlays y badges.
/* Mismo HTML, 3 versiones visuales */
.wrapper.normal { display: block; }
.wrapper.flex { display: flex; gap: 12px; }
.wrapper.grid { display: grid; grid-template-columns: 1fr 1fr; gap: 12px; }

7.6. Errores sutiles de alto impacto (checklist de examen)

  • Unidades olvidadas: width: 100 en vez de 100px.
  • Clase mal escrita: .contenedor vs .container.
  • Descendencia incorrecta: .menu li a no coincide con HTML real.
  • Radio sin name comun: se pueden marcar varios.
  • Altura 100% sin contexto: el padre no tiene altura definida.
  • Float sin clear: pie de pagina sube y se monta.
  • Overflow accidental: ancho fijo + padding + border sin box-sizing: border-box.
  • Especificidad bloqueando cambios: un selector con id pisa todo.
  • Cierre de etiqueta ausente: rompe el DOM y cascada posterior.

7.7. Estructura minima recomendada para tareas largas

<body>
    <div class="page">
        <header class="site-header">...</header>

        <nav class="site-nav">
            <ul class="menu">...</ul>
        </nav>

        <main class="layout">
            <section class="content">...</section>
            <aside class="sidebar">...</aside>
        </main>

        <footer class="site-footer">...</footer>
    </div>
</body>

Esta base evita el 80% de errores estructurales en practicas donde el CSS es pesado.

7.8. Estrategia de practica (para ejercicios de horas)

  1. Replica estructura HTML sin estilo hasta que el orden semantico sea correcto.
  2. Activa layout principal (flex o grid) antes de colores y tipografia.
  3. Resuelve cada bug con una hipotesis: "esta regla no aplica por X".
  4. Valida estados: hover, focus, mobile y contenido largo.
  5. Al final, limpia selectores redundantes y baja especificidad.

7.9. Mini Exam Lab (5 bugs sutiles)

Intenta resolver cada caso sin ver la solucion. Luego abre el bloque de respuesta y compara tu razonamiento.

Lab 1: menu con separadores rojos

/* BUG */
.menu a {
    border-right: 2px solid red;
    padding: 0 12px;
}

/* Resultado: separadores desalineados y dificil quitar el ultimo */
Ver solucion 
.menu {
    display: flex;
    list-style: none;
    margin: 0;
    padding: 0;
}

.menu li {
    border-right: 2px solid red;
    padding: 0 12px;
}

.menu li:last-child {
    border-right: none;
}

El separador debe vivir en li, que representa cada item estructural.

Lab 2: formulario que "no valida bien"

<label>Email</label>
<input id="correo" type="email">

<input type="radio" name="ingles1" value="1">
<input type="radio" name="ingles2" value="2">
Ver solucion 
<label for="correo">Email</label>
<input id="correo" name="correo" type="email" required>

<input type="radio" name="ingles" value="1">
<input type="radio" name="ingles" value="2">

El for conecta label-input y los radios deben compartir name para exclusividad.

Lab 3: un div faltante rompe toda la maqueta

<div class="page">
    <main class="content">...</main>
    <aside class="sidebar">...</aside>
</div>

/* CSS esperado */
.layout { display: flex; }
.layout > .content { flex: 1; }
Ver solucion 
<div class="page">
    <div class="layout">
        <main class="content">...</main>
        <aside class="sidebar">...</aside>
    </div>
</div>

Faltaba el contenedor .layout, por eso los selectores y el flex no se aplicaban.

Lab 4: absolute fuera de sitio

.badge {
    position: absolute;
    top: 0;
    right: 0;
}
Ver solucion 
.card {
    position: relative;
}

.badge {
    position: absolute;
    top: 0;
    right: 0;
}

Sin padre posicionado, absolute toma como referencia el viewport o ancestro incorrecto.

Lab 5: overflow fantasma en columnas 50/50

.col {
    width: 50%;
    padding: 16px;
    border: 1px solid #000;
    float: left;
}
Ver solucion 
* {
    box-sizing: border-box;
}

.col {
    width: 50%;
    padding: 16px;
    border: 1px solid #000;
    float: left;
}

Sin border-box, padding y border se suman al 50% y causan desborde.

7.10. Hard Mode Lab (5 trampas de examen)

Estos casos son mas traicioneros: el bug suele estar en especificidad, contexto de apilamiento, o en reglas que parecen correctas pero no aplican por estructura.

Hard 1: guerra de especificidad

/* BUG: no cambia color aunque editas .btn */
.btn { color: white; }
#panel .toolbar .btn { color: yellow; }

/* HTML real */
<div id="panel">
    <div class="toolbar">
        <a class="btn" style="color: red">Click</a>
    </div>
</div>
Ver solucion 
/* Quita el inline style del HTML (preferido) */
<a class="btn">Click</a>

/* o define una regla de estado/control sin depender de inline */
#panel .toolbar .btn { color: yellow; }

La trampa es el style="..." inline (especificidad superior). No era un problema de la clase base.

Hard 2: pseudo-selector en elemento equivocado

/* BUG: la raya roja sale mal o no sale */
.menu a:not(:last-child) {
    border-right: 2px solid red;
}
Ver solucion 
.menu li:not(:last-child) {
    border-right: 2px solid red;
}

.menu li a {
    display: block;
    padding: 0 12px;
}

:last-child debe evaluarse sobre los li, no sobre los a internos.

Hard 3: clearfix correcto pero contenedor sigue roto

.wrapper::after {
    content: "";
    clear: both;
}

.col { float: left; width: 50%; }
Ver solucion 
.wrapper::after {
    content: "";
    display: table; /* o display:block */
    clear: both;
}

Sin display, el pseudo-elemento puede no participar como esperas y el clear no hace efecto util.

Hard 4: media query "correcta" que nunca dispara

/* BUG */
@media (max-width: 768) {
    .layout { grid-template-columns: 1fr; }
}
Ver solucion 
@media (max-width: 768px) {
    .layout { grid-template-columns: 1fr; }
}

Faltaba la unidad px. Es un error minimo que invalida la condicion.

Hard 5: z-index no funciona (stacking context)

.modal {
    z-index: 9999;
}

.card {
    transform: translateZ(0);
}
Ver solucion 
.modal {
    position: fixed; /* o absolute/relative segun caso */
    z-index: 9999;
}

/* Evitar stacking contexts innecesarios en ancestros */
.card {
    transform: none;
}

z-index solo opera en elementos posicionados y puede quedar encerrado por contextos de apilamiento creados por transform, opacity, etc.

🎓 Resumen Final

¿Cuándo usar cada técnica?

Flexbox Layouts modernos, barras de navegación, centrado, distribución de items
Grid Layouts complejos de dos dimensiones, galerías, dashboards
Float Envolver texto alrededor de imágenes, código heredado

Buenas Prácticas

  • ✅ Usa flexbox para layouts lineales y modernos
  • ✅ Usa grid para layouts bidimensionales complejos
  • ✅ Evita float para layouts principales (usa flexbox/grid)
  • ✅ Siempre usa box-sizing: border-box en todos los elementos
  • ✅ Semantiza tu HTML con etiquetas apropiadas (<header>, <nav>, etc.)
  • ✅ Usa formularios nativos de HTML, no divs personalizados
  • ✅ Valida inputs en el servidor, no solo en el cliente
  • ✅ Usa Canvas para gráficos dinámicos, SVG para estáticos
