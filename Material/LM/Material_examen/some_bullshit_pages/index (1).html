Semantics & Common Elements

Semantic HTML communicates meaning to browsers, crawlers and assistive tech. Use the right element for the job.

Document Components

<header>
Header for a section, typically contains heading, logo, navigation.
<nav>
Navigation landmarks. Put a single set of main links for screen readers.
<main>
Main content of the document; only one per page.
<article>
Self-contained composition — could be syndicated (blog post, news item).
<section>
Logical grouping of content with a heading.
<aside>
Related tangential content (sidebars, callouts).
<footer>
Footer of a section or document.

Text-level semantic tags

Use <strong> and <em> for meaning, not styling. Use <mark> for highlight, <time> for dates, <code> for code.

Example: Important: the em emphasizes speech stress. A timestamp: December 9, 2025. Inline code sample: SELECT * FROM users;.

Selectors, Specificity & Pseudo stuff

Selector types — examples

  • * (universal)
  • div (type selector)
  • .class (class selector)
  • #id (id selector)
  • [attr] and [attr="value"] (attribute selectors)
  • div > p (child selector), div p (descendant)
  • h2 + p (adjacent sibling), h2 ~ p (general sibling)
  • Pseudo-classes: :hover, :focus, :nth-child(2n)
  • Pseudo-elements: ::before, ::after, ::first-letter

Specificity quick rule: inline style (1000) > id (100) > class/attribute/pseudo-class (10) > element/pseudo-element (1). Add !important to force override (use sparingly).

Practical: Specificity override demo

This text is styled by multiple rules; see the CSS to understand which wins.

/* order in CSS file */
.spec-demo__text { color: #ddd; }        /* element+class (specificity 11) */
#unique .spec-demo__text { color: #0b84ff;} /* id + class (specificity 111) */
.spec-demo__text:hover { color: #ffb000;} /* pseudo-class (specificity 11) */
div .spec-demo__text { color: #f8f8f8;}  /* descendant (specificity 11) */
/* inline style would beat the above */
Notes

Specificity examples: an ID wrapper will often win over class selectors. Use classes to keep specificity low and predictable.

Float lab (old-school)

Float approach

Problem scenario: naive inline image placement can push text under the image on narrower viewports.

Floats remove the element from the normal flow horizontally while letting text wrap. The container must be cleared to avoid collapse.

CSS used:

.float-img { float:right; width:140px; margin-left:12px; }
.float-wrap::after { content:""; display:block; clear:both; }

Outcome: good wrapping on wide screens. On narrow screens the image may still push content; use media queries to switch layout.

Key points

  • Floats are great for text wrap, but require clearfix or overflow:auto on the container.
  • Floats are not ideal for complex responsive layouts.

Inline-block approach

This uses display:inline-block on both sides. Text will align to baseline unless vertical-align is changed.

Inline-block respects widths but is sensitive to whitespace in HTML (you may see small gaps).

Key points

  • Good for simple side-by-side content without flexbox.
  • Watch for white-space gaps and baseline alignment.

Flexbox approach (recommended for 1D layout)

Use flex when: you want reliable horizontal alignment and predictable wrapping. Flex can reorder items (order property) and align them vertically (align-items).

CSS used:

.container{ display:flex; align-items:flex-start; gap:12px; flex-wrap:wrap;}
.container img{ max-width:100%; height:auto; }

Grid approach (recommended for 2D layout)

CSS Grid lets you define explicit rows and columns. Great for complex responsive layouts.

.grid { display:grid; grid-template-columns: 220px 1fr; gap:12px; }
@media (max-width:700px) { .grid { grid-template-columns: 1fr; } }

Switch to a single column on small screens using media queries.

Positioning: static, relative, absolute, fixed, sticky

static (default) — normal flow. relative — offsets the element without removing it from flow. absolute — removed from flow, positioned relative to nearest positioned ancestor. fixed — relative to viewport. sticky — behaves like relative until scroll threshold, then fixed.
