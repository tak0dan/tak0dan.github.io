Paragraph selectors & specificity playground

The five paragraphs below are identical in content but targeted by different selectors. Open the code toggles to see what selector is applied and why.

Type selector: p

This is a basic <p> type selector demo — styled by the generic p rule in the CSS. It is intentionally simple so you can inspect "element-only" rules and their limitations.



This is a basic <p> type selector demo — styled by the generic p rule.


/* CSS */
p { font-size:16px; line-height:1.6; color:#e6eef8; }
Type selectors are low-specificity. They are useful for base styling but easy to override. Use them for defaults only.

ID selector: #hero

This paragraph has an id="hero" and is targeted by an ID selector. ID selectors carry a higher specificity score than classes and element selectors.



This paragraph has an id="hero" and is targeted by an ID selector.


/* CSS */
#hero { background: linear-gradient(90deg,#0b84ff22,#07a3ff22); padding:10px; border-left:4px solid #0b84ff; }
IDs are powerful but brittle. They increase specificity so much that overrides become harder. Prefer classes for reusable styles; use IDs for unique components or anchor targets.

Pseudo-class: p:first-child

This paragraph is highlighted because the stylesheet uses p:first-child to target the first paragraph inside a container. Pseudo-classes express structural or behavioral states.

/* CSS */
/* first-child targets the first child element of a parent. If you need the first paragraph specifically, use p:first-of-type instead. */
.container p:first-child { font-weight:700; color:#ffd166; }
Pseudo-classes like :first-child select elements by document structure or state. For "first of type" use :first-of-type. For dynamic states use :hover, :focus.

Pseudo-class + attribute: p:last-child & [data-demo]

This paragraph is targeted by p:last-child and an attribute selector [data-demo="last"]. Attribute selectors are very handy for marking semantics in HTML without creating extra classes.

/* CSS */
/* last-child selects the last child of its parent. Attribute selector matches nodes with the attribute. */
p:last-child { border-top:1px dashed rgba(255,255,255,0.06); padding-top:8px; }

p[data-demo="last"] { font-style:italic; color:#a1e3a1; }
Combining selectors increases specificity and expressiveness. Attribute selectors are great for JS hooks and for when you want semantic flags in markup without extra classes.

Structural selectors: :nth-child() & classes

This paragraph is an example of using :nth-child(3) or :nth-of-type(odd) to select repetitive patterns. Useful for zebra-stripes or alternating styles.

/* CSS */
/* nth-child selects by index inside parent. nth-of-type is safer for selecting by tag type. */
.container p:nth-child(3) { transform: translateX(2px); }
.container p:nth-of-type(odd) { background: rgba(255,255,255,0.02); }

/* Classes are the most recommended approach for reusable styling. */
.para { padding:8px; border-radius:6px; }
Structural selectors are powerful for patterns. They are fragile if document structure changes. Use classes when you want stability.

Alignment & layout mini-lab

Float vs Flex vs Grid

Float example: the image is floated to the right while text wraps around it. On narrow screens the text may drop below the image.

/* CSS float example */
.float-thumb { float:right; width:240px; margin-left:12px; }
.float-wrap::after { content:''; display:block; clear:both; }

Flex example: reliable horizontal layout. Use flex-wrap and min-width to control wrapping behavior on small viewports.

/* CSS flex example */
.flex-container { display:flex; gap:12px; align-items:flex-start; }
.flex-thumb { flex:0 0 240px; }
.flex-content { flex:1; min-width:220px; }

Grid example: explicit columns. Change grid-template-columns based on breakpoints to stack or expand columns.

/* CSS grid example */
.grid { display:grid; grid-template-columns:240px 1fr; gap:12px; }
@media (max-width:700px) { .grid { grid-template-columns:1fr; } }

Animations, transitions & pseudo-elements

Hover transition + keyframe demo

/* CSS transition */
.btn { transition: transform 160ms cubic-bezier(.2,.9,.3,1), box-shadow 160ms; }
.btn:hover { transform: translateY(-4px); box-shadow: 0 8px 24px rgba(11,132,255,0.12); }

/* keyframes */
@keyframes pulse { 0% { transform: scale(1) } 50% { transform: scale(1.03) } 100% { transform: scale(1) } }
.pulse { animation: pulse 2.5s infinite; }
Transitions are for single-state changes; keyframes for repeating or multi-step animations. Keep animations subtle and provide prefers-reduced-motion alternatives.
