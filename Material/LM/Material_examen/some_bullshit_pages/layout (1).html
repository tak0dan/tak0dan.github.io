Layout Lab — Learn how things sit on a page

From old-school floats to modern grid — everything explained step-by-step.

Quick Reference: What each layout mode means

  • Block: takes full width, starts on a new line. Examples: <div>, <p>.
  • Inline: sits inside text, only as wide as content. Examples: <span>, <a>.
  • Inline-block: like inline but can have width/height.
  • Floats: old way to let text wrap around elements. Requires clearing.
  • Flexbox: 1-dimensional layout for rows or columns — great for toolbars.
  • Grid: 2-dimensional layout — rows AND columns; powerful for full pages.
  • Positioning: place elements explicitly (absolute/fixed/sticky/relative).

Tip: Use Flex for simple rows, Grid for complex 2D layouts, and avoid floats unless you need text wrapping.

Floats & Clearing

Floats were originally for wrapping text around images. They remove an element from normal inline flow horizontally but keep it in the flow vertically.

This paragraph wraps around the floated image. If the image is too wide, the text may drop below it on narrow screens.

To prevent the parent collapsing you must clear the float using clearfix, overflow:auto, or display:flow-root.

/* CSS */
.float-thumb { float:right; width:160px; margin-left:12px }
.float-wrap::after { content:''; display:block; clear:both }
/* Alternative: .float-wrap { display:flow-root; } */
Important: floats can cause layout collapse (parent height 0). Use clearfix or flow-root to create a block formatting context that contains floats.

Block Formatting Context (BFC)

BFC is a concept that controls how boxes interact and clear floats. Creating a new BFC fixes many float bugs.

Even if the child is floated, the container correctly measures height because it has a BFC.

/* CSS */
.bfc-container { overflow:auto; }
/* Other ways: display:flow-root; contain:block; */
Exam tip: Know at least 2 ways to create a BFC: overflow (not visible), display:flow-root (clean), or float on the container (legacy).

Positioning: static, relative, absolute, fixed, sticky

Positioning changes how elements are placed relative to normal flow, parent elements, or the viewport.

Parent (position: relative)

Absolutely positioned child (top-right)
/* CSS */
.pos-parent { position:relative; height:120px; background:#f7f7f7 }
.pos-child { position:absolute; right:10px; top:10px; background:#4a90e2; color:white; padding:6px }
Remember: absolute is positioned relative to the nearest positioned ancestor (non-static). fixed is relative to the viewport. sticky toggles between relative and fixed depending on scroll.

Responsive techniques & media queries

Use media queries, fluid units (%, vw, rem), and clamp() for scalable layouts.

Resize the window to see layout switch at the breakpoint in layout.css.

/* CSS */
.container { display:grid; grid-template-columns: 300px 1fr }
@media (max-width:800px) { .container { grid-template-columns: 1fr } }
Mobile-first approach: write styles for small screens first, then add min-width media queries to expand layout for larger screens.

Common pitfalls & exam traps

  • Using width:100% on children inside padding-heavy containers (box-sizing matters).
  • Forgetting to set box-sizing:border-box leads to confusing width maths.
  • Absolute positioning removes element from flow; it may overlap content unexpectedly.
  • Not using responsive units: avoid fixed px sizes for major layout widths.
Exam tip: mention a fallback for older browsers if the question hints at compatibility (e.g., graceful fallback when grid is not supported).
