CSS SELECTORS — COMPLETE VISUAL PLAYGROUND

Every selector. Every trick. Fully explained.

1) Type Selectors

Type selectors match HTML elements directly: p, h1, li, etc.

This paragraph is styled by the P selector.

Another paragraph also styled by the P selector.

/* selectors.css */
p {
  color: blue;
  font-weight: bold;
}

2) Class Selectors

Class selectors start with a dot: .example. They’re reusable and low specificity.

I am targeted by .demo-class
Me too — same class, same style.
/* selectors.css */
.demo-class {
  background: #e0f0ff;
  border-left: 4px solid #4a90e2;
}

3) ID Selectors

IDs begin with # and should be UNIQUE per page. Very high specificity.

This element uses an ID selector.

/* selectors.css */
#important-message {
  color: white;
  background: #4a90e2;
  padding: 10px;
}

4) Attribute Selectors

Match elements by attributes: [data-kind], [type="email"], etc.

This has data-kind="note"

This has data-kind="warning"

/* selectors.css */
[data-kind] {
  padding: 6px;
}
[data-kind="note"] {
  background: #fffbd1;
}
[data-kind="warning"] {
  background: #ffe2e2;
}
input[type="email"] {
  border: 2px solid #4a90e2;
}

5) Combinators

Examples: div p descendant, div > p direct child,h2 + p adjacent sibling, h2 ~ p general sibling.

I am a CHILD (div > p)

I am a DESCENDANT (div p)

Heading

I appear right after heading (h3 + p)

I also follow the heading but not immediately (h3 ~ p)

/* selectors.css */
.parent-box > p.child { background: #d4ebff; }
.parent-box p.descendant { background: #e9f5ff; }
h3 + p.adjacent { border-left: 3px solid #4a90e2; }
h3 ~ p.general { opacity: 0.7; }

6) Pseudo-classes

I am the FIRST child

I am the SECOND child

I am the THIRD child

/* selectors.css */
.hover-btn:hover {
  background: #4a90e2;
  color: white;
}
.nth-demo:first-child {
  border-left: 4px solid #4a90e2;
}
.nth-demo:nth-child(2) {
  background: #e9f2ff;
}

7) Pseudo-elements

This paragraph uses ::before and ::after.

/* selectors.css */

.before-after-demo::after {
