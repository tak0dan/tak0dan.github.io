Alignment Lab — fix that image-on-right, text-below bug

Four approaches, with pitfalls and fixes. Resize your browser to test.

1) Float

What happens: Text wraps around the floated image on wide screens; on narrow screens the image may be wider than the text column and the text drops below the image.

Why it goes wrong: float removes element from flow horizontally but not vertically; if the image width is larger than available inline space, block-level text will break under it.

How to fix: use percentage widths or media queries to switch to stacked layout; apply clearfix (done above) or set overflow:auto on wrapper to contain floats.

CSS used: float:right; and a clearfix ::after{clear:both}. Responsive fix: on @media remove float (float:none;display:block;).

2) Inline-block

What happens: Inline-block keeps elements in flow and side-by-side if there's space; on narrow screens the second block wraps below. Small whitespace in HTML can introduce gaps between inline-block elements.

Why it goes wrong: If your widths add up to >100% (or include margins), the second element wraps and appears below the first.

How to fix: reduce widths, remove whitespace between inline-block nodes, or use font-size:0 on the parent then restore font-size for children; but modern answer: use flex or grid.

Inline-block is simple but fragile — spacing, baseline alignment and width calculation bite you. For robust responsive layouts use flex or grid.

3) Flexbox (recommended)

What happens: Flex distributes space. The image stays at its basis width; content takes remaining space. On small screens, flex-wrap allows content to wrap under the image or the image to shrink if allowed.

Why it fails: If you set a fixed image width and the container is narrower than image+text minimums, content will wrap under the image. Use min-width, flex-basis and media queries appropriately.

How to fix: set min-width on content or allow image to shrink flex:1 1 200px, or switch to column direction on small screens with media query.

Flex is the practical choice for 1-dimensional layouts: row or column. Use flex-wrap, min-width, and media queries for robust responsive behavior.

4) CSS Grid (recommended for complex layouts)

What happens: Grid gives explicit control: you map the image to a column and the text to another column. If the grid column sizes exceed container width, items stack when you change grid-template with media queries.

Why it fails: Forgetting to change grid-template on small screens will keep image and content cramped or cause text to overflow. Grid does not auto-wrap like flex unless you define it.

How to fix: Use grid-template-columns: 240px 1fr; and a media query to switch to 1fr (one column) under a breakpoint.

Grid is best when layout is 2-dimensional. It’s explicit, powerful and predictable — perfect for exam answers.

Pro tip: the correct answer often expected in exams: use flex or grid with media queries to switch to stacked layout. Floats are acceptable historically, but mention their pitfalls and the necessary clearfix.
